## ⏱️ How to Use This Practice Sheet
1. **Time Box**: Allocate exactly 45 minutes for whiteboarding and drawing layout diagrams.
2. **Focus on Requirements**: Always detail functional/non-functional bounds and QPS limits before drawing blocks.
3. **Identify Bottlenecks**: Dig deep into hot shards, caching strategies, and data consistency models.
4. **Post-mortem Notes**: Document your specific mistakes and feedback inside the problem's study notes sheet.
